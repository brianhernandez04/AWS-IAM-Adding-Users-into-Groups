# AWS IAM: Adding Users into Groups

This article will guide you on how to add specific Users into Groups in AWS leveraging Identity and Access Management service.

> **Prerequisites:** Previously created Users & Groups.

---

## Steps

### Step 1. Navigate to User Groups

We will start off by going into the User Groups tab and selecting the Group you would like to add a user in. In this example, it is the EC2-Support group.

<img width="1100" height="268" alt="1_bTSmrmT1c-oj71esOV-czA" src="https://github.com/user-attachments/assets/537b39ca-8923-464c-8f71-b1ae4c7b8ccb" />

---

### Step 2. Add Users to the Group

Once you are inside of the group settings > Please click on the "Add Users" button on the bottom right of the Users section.

<img width="800" height="307" alt="1_8XlOOCJ_LsiSXX3pzKqaQA" src="https://github.com/user-attachments/assets/b64c2efc-fbb4-4809-bf16-73e291ab4532" />

---

### Step 3. Select your User(s)

You will be prompted to select the specific user(s) that you had previously created. Be sure to click on the checkbox for the users that you want and click on the add users button. In this example we are using user-2.

<img width="1100" height="374" alt="1_KJyDaSsnPsI9ZPA1QvmP3A" src="https://github.com/user-attachments/assets/8d719dd8-9464-49cb-8f50-e8f54c534c14" />

---

### Step 4. Confirm the User was Added

Back to the User Groups dashboard, you should see a green banner at the top indicating you added the user succesfully. You should also see the user you just added underneath the Users section of the Groups dashboard.

<img width="1100" height="523" alt="1_Nvt6YJRzN0sK_T0FDbMbQw" src="https://github.com/user-attachments/assets/a2a39d6d-886d-4b7e-ae3b-c8a04e84a5cd" />

---

### Step 5. Double-Check the User's Group Assignment

You can confirm/double-check the user was added successfully by going to the Users tab on the left hand side > Selecting the user you just added in the group > Click on the groups icon > You will see the group that the user is in and a brief overview of the attached policies to the group.
